Welcome to the May 2026 edition of "What's New in krkn-chaos," where we bring you the latest and greatest developments from our ecosystem. This month, we have a host of new features, improvements, and bug fixes to enhance your chaos engineering experiences on Kubernetes.

## 🚀 New Features

- **Auto-Generate Release Notes**: The krkn-operator now supports automatic generation of release notes using conventional commits, streamlining the release process.
- **Cluster Terminal Feature**: A new cluster terminal feature has been added to the krkn-operator, providing users with seamless terminal access to their clusters.
- **Auth Token Support**: krkn-ai introduces auth token support for health check URLs, enhancing security and access control during health checks.

## 🛠️ Improvements

- **Multi-Platform Image Builds**: krkn-hub now supports multi-platform image builds for both amd64 and arm64, broadening compatibility and deployment options.
- **Rollback Scenarios**: Enhanced rollback scenarios for krkn-operator and krknctl implementations are now available, providing more robust rollback capabilities.
- **Python 3.11 Alignment**: Documentation has been updated to align Python requirements with version 3.11, ensuring compatibility and performance improvements.

## 🐛 Bug Fixes

- **Race Condition Fixes**: Resolved race conditions in krknctl's Attach function and krkn-ai's node interface discovery, leading to more stable operations.
- **Prometheus Handling**: Fixed handling of empty Prometheus values in fitness calculations within krkn-ai, improving accuracy in chaos experiments.
- **Error Logging and Handling**: Improved error logging for Alibaba requests and fixed handling of missing kubeconfig and Prometheus tokens in krkn-ai.

We hope these updates enhance your experience with krkn-chaos. Stay tuned for more features and improvements in the coming months!
